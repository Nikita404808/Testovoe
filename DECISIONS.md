# DECISIONS

1. Использован SQLite через встроенный `node:sqlite`.
   Это позволило сделать проект без внешних npm-зависимостей и упростить локальный запуск.

2. Выбран сервер на стандартном `http` Node.js вместо фреймворков.
   Для тестового задания этого достаточно, а поведение маршрутов и статусов полностью контролируется в коде.

3. Переход `assigned -> in_progress` реализован атомарным SQL `UPDATE ... WHERE status='assigned'`.
   Это обеспечивает race-safe поведение: при параллельных запросах только один обновляет строку.

4. Логику работы с заявками вынесена в отдельный слой `Store`.
   Роуты отвечают за HTTP/авторизацию, а бизнес-правила и переходы статусов централизованы в одном месте.

5. Авторизация сделана сессиями в памяти и пользователями из сидов.
   Для тестового стенда этого достаточно: не требуется внешняя auth-система, вход выполняется по `name/password`.

6. Добавлен audit log (`request_events`).
   Это не обязательный пункт, но помогает отслеживать жизненный цикл заявки и упрощает отладку.
